# Owner Workspace Density

## Goal
Make the owner area feel compact and professional on desktop while staying clear and usable.

## Desktop Rules
- prefer a slim sidebar or navigation rail plus a tighter main workspace
- reduce oversized spacing and heading scale across dashboard, editor, links, tags, and search
- keep more real content above the fold
- secondary panels should support the main workflow rather than compete with it
- if a dashboard tools/promotional block steals space from Notes on desktop, remove it or demote it below the main notes surface
- if left navigation already exposes Links, Tags, and Search clearly, do not repeat them inside the Notes header area

## Secondary Surface Rules
- owner links should prioritize the saved-items list as the main growing surface
- static capture forms should not sit beside long scrollable lists when that creates wasted vertical space

## Surface Rules
- dashboard lists should feel denser than the public homepage
- forms should use smaller but still comfortable field spacing
- editor and preview surfaces should feel like tools, not oversized marketing cards

## Anti-Goals
- do not collapse into enterprise-dashboard visual noise
- do not preserve the current large hero/panel proportions on desktop
- do not solve density by shrinking touch targets below comfortable desktop/mobile use

# Knowledge Studio Visual System

## Goal
Refresh Minakeep into an elegant knowledge-studio product that feels close to the reference in density and polish without becoming a visual clone.

## Core Direction
- editorial typography over generic dashboard defaults
- dense but calm information layout
- strong hierarchy for titles, summaries, metadata, and tags
- one coherent visual language across public and private surfaces

## Public Surfaces
- homepage should feel like a curated research journal, not a landing-page template
- public note pages should prioritize reading comfort while still surfacing useful metadata

## Private Surfaces
- the owner area stays notes-first
- lists, forms, previews, and metadata panels should feel premium and deliberate rather than scaffold-like
- AI status should be visible but visually secondary to authored content
- AI status treatment should stay consistent across dashboard cards, the note editor, links, tags, and search rather than changing its hierarchy route by route

## Anti-Goals
- do not reproduce the reference layout one-for-one
- do not turn the app into a neon metrics dashboard
- do not add chat-assistant visual patterns in this wave

# DESIGN.md

## Purpose
Capture the top-level design direction for Minakeep and point to the deeper design docs that define the real constraints.

## Current Design Themes
- one repository that serves both capture and selective publication
- small, explicit route shells over generic dashboard sprawl
- quiet public reading surface, practical private writing surface
- elegant knowledge-studio density across the whole app rather than a generic admin look
- AI enrichment should feel embedded into save/review flows, not like a separate assistant product
- structure the repo so future feature waves can land as narrow vertical slices

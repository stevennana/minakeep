# Public surface human design reset

## Goal
Refine the public Minakeep experience so published notes and links feel content-led, bespoke, and intentionally designed rather than generically AI-generated.

## Trigger / Entry
An anonymous visitor opens the public homepage or a public note page.

## User-Visible Behavior
- Public chrome such as the topbar, archive counter, and collapsed search row recedes behind the published content.
- The public homepage feels more like a living archive than a boxed app shell.
- Desktop showroom layout feels more dynamic than the current two-column impression.
- Typography across public surfaces feels calmer, smaller, and more human-made.
- Tag chips fit their text correctly and remain legible on both note and link cards.
- Public note pages inherit the same calmer hierarchy and bespoke typography rules.

## Validation
- The public surfaces feel content-first rather than chrome-first.
- The homepage and public note page share one coherent but non-generic visual language.
- Tags, headings, and metadata feel more refined than the current shipped tranche.
- `npm run verify` passes.

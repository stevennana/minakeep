# Knowledge studio UI refresh

## Goal
Refresh the full Minakeep UI into an elegant, reference-inspired knowledge-studio experience across public and private surfaces.

## Trigger / Entry
The owner or a public reader visits any core Minakeep surface.

## User-Visible Behavior
- The public homepage, public note pages, login, and private owner routes share one coherent visual system.
- The refreshed UI feels dense and polished without becoming a clone of the reference product.
- Authored content remains primary, while AI metadata and operational status remain visible but secondary.
- The redesign preserves the existing route model and information architecture.
- Owner note and link cards keep one consistent AI-status treatment across the dashboard, editor, links, search, and tags routes.

## Validation
- Public and private surfaces share the new visual language consistently.
- Responsive behavior remains solid on mobile and desktop.
- The redesign does not regress existing note, publishing, link, tag, or search flows.
- `npm run verify` passes.

# References

This directory stores project reference material intended to help agents work inside the repository without relying on external context.

If the user provides references, analyze them and keep the useful project-specific information here so future work can rely on it.

This applies to:

- local reference files the user points at
- online references the user links
- API docs, reference repos, product inspirations, and architecture examples

Do not just mention those references in chat and drop them.
Turn them into durable repo-local reference notes that preserve the information needed for future implementation and next-version planning.

Add reference files here when they materially improve agent legibility:

- framework references
- API contracts
- request/response shapes
- local conventions

When adding a reference note:

- summarize the parts that matter to this project
- avoid dumping the whole source verbatim
- record the source location clearly
- connect the reference back to the project’s product, design, architecture, or exec-plans
